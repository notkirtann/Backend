## 1. Project Overview

### Backend Responsibility

Backend developers focus on:

* Data collection
* Data storage
* Data processing

Not responsible for:

* UI design
* Layout or visual elements

For a content platform:

* User data
* Liked videos
* Watch history

This requires:

* **Advanced data modeling**

## 3. Professional Project Structure

### Recommended Structure

```
project-root/
│
├── src/
│   ├── controllers/
│   ├── models/
│   ├── routes/
│   ├── middlewares/
│   ├── utils/
│   └── index.js
│
├── public/
│   └── temp/
│       └── .gitkeep
│
├── .env
├── .gitignore
├── package.json
└── README.md
```

### Why `.gitkeep`?

Git does not track empty folders.
Adding `.gitkeep` allows version tracking.

---

## 4. Environment Configuration

Create:

```
.env
```

Example:

```
PORT=8000
MONGODB_URI=your_db_url
JWT_SECRET=your_secret
```

Never push `.env` to GitHub.

---

## 5. .gitignore Setup

Use a generator or create manually:

```
node_modules
.env
public/temp
dist
```

Purpose:

* Prevent sensitive data exposure
* Reduce repository size

---

## 6. Enable ES Modules

In `package.json`:

```json
{
  "type": "module"
}
```

Now you can use:

```js
import express from "express";
```

instead of:

```js
const express = require("express");
```

## 7. Development Workflow – Nodemon

Install:

```bash
npm install nodemon --save-dev
```

Add script:

```json
"scripts": {
  "dev": "nodemon src/index.js"
}
```

Run:

```bash
npm run dev
```

Benefit:

* Auto-restarts server on file changes

## 8. Code Formatting – Prettier

### Install

```bash
npm install --save-dev prettier
```

### Create `.prettierrc.json`

```json
{
  "singleQuote": false,
  "bracketSpacing": true,
  "tabWidth": 2,
  "semi": true
}
```

### Create `.prettierignore`

```
node_modules
.env
public/temp
```

Benefits:

* Consistent formatting
* Team collaboration
* Cleaner codebase

# Express Backend & Deployment – Summary Notes

## 1. Express.js for Backend

Express is the core framework used to build the backend server.

### Installation

```bash
npm init -y
npm install express
```

### Basic Server Example

```js
import express from "express";

const app = express();
const PORT = 3000;

app.listen(PORT, () => {
  console.log(`Server running on port ${PORT}`);
});
```

---

## 2. Handling Routes

Routes define how the server responds to client requests.

### Example Routes

```js
app.get("/", (req, res) => {
  res.send("Home Page");
});

app.get("/twitter", (req, res) => {
  res.send("Twitter Page");
});

app.get("/youtube", (req, res) => {
  res.send("YouTube Page");
});
```

### Different Response Types

**Plain Text**

```js
res.send("Hello World");
```

**HTML**

```js
res.send("<h1>Welcome</h1>");
```

**JSON**

```js
res.json({
  name: "Kirtan",
  platform: "YouTube"
});
```

## 3. Environment Variables with Dotenv

Sensitive data should not be hardcoded.

### Install

```bash
npm install dotenv
```

### Usage

**.env**

```
PORT=5000
DB_URL=mongodb://localhost:27017/test
API_KEY=secret_key
```

**index.js**

```js
import dotenv from "dotenv";
dotenv.config();

const PORT = process.env.PORT;
```

### Why Use Dotenv?

* Keeps secrets secure
* Easy configuration for different environments
* Required for production deployment

## 5. GitHub Integration

Deployment platforms usually deploy from GitHub.

### Steps

Initialize Git:

```bash
git init
git add .
git commit -m "Initial commit"
```

Push to GitHub:

```bash
git remote add origin <repo-url>
git push -u origin main
```

---

## 6. .gitignore (Important)

Exclude unnecessary and sensitive files.

**.gitignore**

```
node_modules
.env
dist
```

Why?

* Prevents leaking secrets
* Reduces repository size

---

## 7. Live Deployment Process (DigitalOcean)

### Steps Overview

1. Push project to GitHub
2. Connect GitHub repository to DigitalOcean
3. Configure build settings
4. Add environment variables on the platform
5. Deploy the application
6. Get live URL

---

## 8. Environment Variables in Production

Set variables directly in the deployment dashboard:

Examples:

```
PORT=5000
DB_URL=your_production_db
JWT_SECRET=your_secret
```

---

## 9. Verifying Deployment

After deployment:

* Open live URL
* Test routes:

  * `/`
  * `/twitter`
  * `/youtube`
* Confirm responses work correctly

---

## 10. Key Beginner Takeaways

* Express makes backend creation simple
* Always use environment variables for secrets
* Never upload `.env` to GitHub
* GitHub is required for most deployments
* Deployment platforms automate build and hosting
* Testing live routes confirms successful deployment

---
